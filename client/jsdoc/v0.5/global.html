isWebRTCEnabled() → {boolean}

Check if WebRTC is supported by current navigator.

This function checks the presence of an implementation of one of the functions (namely, getUserMedia) defined in W3C specification (http://www.w3.org/TR/mediacapture-streams):

  • navigator.mozGetUserMedia() for Mozilla Firefox.
  • navigator.webkitGetUserMedia() for Google Chrome.

Source:
  • hp-webrtc-api.js, line 5371
Returns:

true if webRTC functionality is supported, false if not.

Type
boolean
Example
    if (isWebRTCEnabled()) {
       console.log('WebRTC is supported');
    } else {
       console.log('WebRTC is NOT supported');
    }

call

WebRTC call event.

WebRTC public event related to a WebRTC call.

Properties:
Name Type Description
state object

The state corresponding to this event:

  • 'TRYING' : The call is trying to connect to destination.
  • 'RINGING' : The call is ringing at destination.
  • 'FORWARDING' : The call is being forwarded.
  • 'QUEUED' : The call is queued.
  • 'EARLY_ANSWERED' : The call is early answered.
  • 'EARLY_TERMINATED' : Early answer is terminated.
  • 'ANSWERED' : The call is answered locally or remotely.
  • 'HANGUP' : The call is hang-up locally or remotely.
  • 'Incoming Message' : An incoming message is notified on the call.
  • 'Incoming Message Status' : An incoming message status is notified on the call.
  • 'Outgoing Message' : An outgoing message will be sent on the call.
message object

The message contains the data payload of the event.

  • When processing an incoming message, the event state is 'Incoming Message' and the event message property contains an object with the following properties:
    • ood : The Operation Opaque Data. It can be a string or an object. This ood is provided by the the Web Application Server. This ood usually represents the origin of the incoming message.
    • msgId : The message identifier.
    • profile : The message profile.
    • payload : The message payload.
  • When processing an incoming message status, the event state is 'Incoming Message Status' and the event message property contains an object with the following properties:
    • msgId : The message identifier for which the status applies.
    • code : The message status code. This code can be:
      • 'SUCCESS' : The message has been successfully sent.
      • 'TIMEOUT' : Timeout when trying to reach the destination.
      • 'UNAVAILABLE' : The destination is unavailable.
      • 'BUSY' : The destination is busy.
      • 'NO_ANSWER' : Destination did not answer.
      • 'NOT_REACHABLE' : Destination is not reachable.
      • 'REJECTED' : The messge is rejected.
      • 'INVALID_RESPONSE' : An invalid response has been received.
      • 'REDIRECTED' : The message is redirected.
      • 'BAD_REQUEST' : The message request is not understood.
      • 'REMOTE_ERROR' : Remote error.
      • 'GLOBAL_ERROR' : Global error.
      • 'INTERNAL_ERROR' : Internal error.
      • 'NOT_AUTHORIZED' : The message is not authorized by the application.
      • 'AUTHORIZATION_FAILED': Failed to check the call authorization with the application.
      • 'IO_ERROR' : Input/Output error.
      • 'TERMINATED' ! The call is forcefully terminated due to local or remote end point termination.
      • 'LIMIT_REACHED' : The message is rejected because some limits have been reached.
  • When processing an outgoing message, the event state is 'Outgoing Message' and the event message property contains an object with the following properties:
    • ood : The Operation Opaque Data. It can be a string or an object. This ood is sent to the Web Application Server. This ood usually represents the destination of the outgoing message.
    • msgId : The message identifier.
    • profile : The message profile.
    • payload : The message payload.

    error

    WebRTC error event.

    WebRTC event indicating that an error occurred.

    An error is first dispatched via an error callback (if provided by the user). Then, it is dispatched to error listeners.

    As a consequence, a user providing an error callback and listening on errors will receive the error twice, unless propagation is stopped at some step of the error processing (i.e., within the user-provided callback).

    Properties:
    Name Type Description
    name string

    The name (type) of the event ('error').
    message string

    An optional error message.
    stopPropagation boolean

    Stop propagation of current error event.
    detail object

    An optional object associated with the error (providing further details on the error).

    • When the value of the message property is 'Incoming Call with MEDIA_ERROR' then the value is the corresponding call instance in 'Destroyed' state.
    • When the value of the message property is 'Call Error' then the following value can be provided:
      • 'TIMEOUT': Timeout when trying to reach the destination.
      • 'UNAVAILABLE': The destination is unavailable.
      • 'BUSY': The destination is busy.
      • 'NO_ANSWER': Destination did not answer.
      • 'NOT_REACHABLE': Destination is not reachable.
      • 'REJECTED': The call is rejected.
      • 'INVALID_RESPONSE': An invalid response has been received.
      • 'REDIRECTED': The call is redirected.
      • 'BAD_REQUEST': The call request is not understood.
      • 'REMOTE_ERROR': Remote error.
      • 'GLOBAL_ERROR': Global error.
      • 'INTERNAL_ERROR': Internal error.
      • 'MEDIA_ERROR': Media error.
      • 'NOT_AUTHORIZED': The call is not authorized by the application.
      • 'AUTHORIZATION_FAILED': Failed to check the call authorization with the application.
      • 'IO_ERROR': Input/Output error.
      • 'TERMINATED': The call is forcefully terminated due to local or remote end point termination.
      • 'LIMIT_REACHED': The call is rejected or forcefully terminated because some limits have been reached.

    event

    WebRTC 'generic' event.

    The object all WebRTC events derive from (media events, call events, signaling channel events, ..etc...).

    Properties:
    Name Type Description
    name string

    The name (type) of the event:

    • 'event' : WebRTC 'generic' event (default value).
    • 'signaling' : WebRTC signaling channel event.
    • 'media' : WebRTC media event.
    • 'ice' : WebRTC ICE event.
    • 'call' : WebRTC call event.
    • 'session' : WebRTC session event.
    stopPropagation boolean

    Stop propagation of current event.

    An event is first dispatched via an event callback (if provided by the user). Then, it is dispatched to listeners as a 'specialized' event (e.g., an event related to a call, a signaling channel, ...), and finally as a 'generic' event.

    As a consequence, a user providing an event callback and listening on both 'specialized' and 'generic' events will receive the event three times, unless propagation is stopped at some step of the event processing (i.e., within the user-provided callback or in the callback provided when registering as listener for a 'specialized' event).

    media

    WebRTC media event.

    WebRTC event related to the media stream of a WebRTC call.

    Properties:
    Name Type Description
    message object

    The message contains the data payload of the event. The message contains these attributes:

    • type [mandatory] : contains the media type ('local' or 'remote').
    • stream [mandatory] : contains the stream chunck.
    • videoEl [optional] : contains the name of the video element where the stream is displayed. Present only for the 'New Attached Stream' and 'Remove Attached Stream' states.
    state string

    The current state of the media stream:

    • 'New Stream' : a new stream contained in the stream attribute of the message has been created.
    • 'New Attached Stream' : a new stream contained in the stream attribute of the message has been created and attached to the video element with the name contained in the videoEl attribute of the message. This state is triggered instead of the 'New Stream' one if a name of a video element has been provided in localVideoEl, inCallLocalVideoEl or remoteVideoEl attributes of the options argument of the connectivity.createCall or call.pickup methods.
    • 'Remove Stream' : the stream contained in the stream attribute of the message has been removed.

    • 'Remove Attached Stream' : the stream contained in the stream attribute of the message has been removed and will be detached from the video element with the name contained in the videoEl attribute of the message. This state is triggered instead of the 'Remove Stream' one if a name of a video element has been provided in localVideoEl, inCallLocalVideoEl or remoteVideoEl attributes of the options argument of the connectivity.createCall or call.pickup methods.

    session

    WebRTC session event.

    WebRTC event related to a WebRTC session (as defined in JSEP specification, see https://tools.ietf.org/html/draft-ietf-rtcweb-jsep-06).

    Properties:
    Name Type Description
    signalingState string

    The session's signaling state:

    • 'initial' : the session's initial state (this state is not defined in JSEP).
    • 'stable' : the session is in a stable state. SDP negotiation succeeded, no re-negotiation pending.
    • 'have-local-offer' : SDP negotiation pending, local offer sent, no remote answer received yet.
    • 'have-remote-offer' : SDP negotiation pending, remote offer received, no local answer sent yet
    • 'have-local-pranswer' : SDP negotiation pending, remote offer received, local provisional answer sent.
    • 'have-remote-pranswer' : SDP negotiation pending, local offer sent, remote provisional answer received.
    • 'closed' : the session is closed (this state is not defined in JSEP).
