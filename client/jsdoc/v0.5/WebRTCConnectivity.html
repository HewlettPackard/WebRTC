<static> WebRTCConnectivity

A WebRTC connectivity to an Internet Signaling Function (ISF).

A WebRTC connectivity is the means to create WebRTC calls.

A connectivity exchanges messages with a remote ISF via a signaling channel. All WebRTC calls created via a connectivity share this signaling channel for their communication with the ISF.

A WebRTC connectivity is an EventEmitter. Typically later on the following events can be notified:

  • event:error : In case of error.
  • event:signaling : In case of connectivity state changes and incoming calls.
  • event:event : If the propagation of the previous events has not been stopped.

<static> createCall(ood, options) → {object}

Creates a WebRTC call.

This method creates a WebRTCCall instance for outgoing call purpose but the call is not initiated. The place() method of the WebRTCCall instance can to be called for initiating the outgoing call. The destroy() method of the WebRTCCall instance can to be called for deleting the call object.

Parameters:
Name Type Description
ood object

the optional Operation Opaque Data. It can be a string or an object

It may be a simple string (eg. an URI) or a complex data object. The content of the ood is sent to the Web Application Server in order to be resolved. This operation is way to hide real user information, so you can place any information you need and it can be resolved by the Web Application Server. Usually this information identifies the destination of an outgoing call.

options object

Optional WebRTC call configuration parameters (see call default configuration).

This configuration may be partial. Parameters provided in this parameter will overwrite default values.
