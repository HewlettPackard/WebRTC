<static> WebRTCCall

A WebRTC call.

A WebRTC call allows to either:

  • Place an outgoing call
  • Answer an incoming call
  • Refuse an incoming call

A WebRTC call is an EventEmitter. Typically later on the following events can be notified:

  • event:error : In case of error.
  • event:call : Regarding call progess and hang-hp from the other call party.
  • event:fsm : Regarding the finite state machine (FSM) of the WebRTC call.
  • event:ice : Regarding ICE negotiation for the WebRTC call.
  • event:media : Regarding the media stream of the WebRTC call.
  • event:session : Regarding the WebRTC session of the WebRTC call.
  • event:event : If the propagation of the previous events has not been stopped.

<protected> WebRTCCall default configuration

Default WebRTC call configuration.

  • localVideoEl: {string} The id of the HTML video tag that will hold the local video stream (at call setup). No default value.
  • inCallLocalVideoEl: {string} The id of the HTML video tag that will hold the local video stream (once established). No default value.
  • remoteVideoEl: {string} The id of the HTML video tag that will hold the remote video stream (once established). No default value.
  • mediaStreamConstraints: media stream constraints, as defined in http://dev.w3.org/2011/webrtc/editor/getusermedia.html#dictionary-mediastreamconstraints-members
    • audio: Set to true, by default.
    • video: Set to true, by default.

Returns the Ood of the call object.

Source:
  • hp-webrtc-api.js, line 4741
Returns:

The Operation Opaque Data (ood) of the call.

It is a string (eg. an URI). For ougoing call this ood is sent to the Web Application Server (this is the 1st argument of the WebRTCConnectivity.createCall method). For incoming calls this ood is provided by the the Web Application Server. This ood usually represents the other party of the call. The Web Application Server has the oportunity to hide or not the effective identity of the other call party.

<static> getState() → {string}

Returns the state of the call object.

Source:
  • hp-webrtc-api.js, line 4780
Returns:

The state of the call which is one of:.

  • 'Ready' : The call has been created and is ready to be placed.
  • 'Active' : The call is active (placed for an outgoing call or pickedup for an incoming call)
  • 'Error' : The outgoing call has failed to setup (after the place operation).
  • 'Hangup' : The incoming call has been refused or the call has been hang-up by one call party.
  • 'Destroyed' : The call has been destroyed.

string
